# IndicoInk V1 Implementation Plan

## 1. Project Baseline

- [ ] Record Electron `42.3.2` as the initial pinned version and add a short supported-version upgrade policy. Verify the documented version matches `package-lock.json`.
- [ ] Scaffold an Electron Forge app with Vite, TypeScript, React, and npm. Verify `npm start` opens a Windows desktop window.
- [ ] Enable `contextIsolation`, renderer sandboxing, and `nodeIntegration: false`. Verify Electron security warnings do not appear at startup.
- [ ] Add a typed, method-specific preload API with no generic IPC send method. Verify the renderer can call one harmless app-info method.
- [ ] Add ESLint, Prettier, and strict TypeScript checks. Verify `npm run lint`, `npm run format:check`, and `npm run typecheck` pass.
- [ ] Add Vitest and one renderer unit test. Verify `npm test` passes.
- [ ] Add a Playwright Electron smoke test that launches and closes the app. Verify `npm run test:e2e` passes on Windows.
- [ ] Add `npm run verify` to run formatting, linting, type checks, unit tests, build, and the smoke test. Verify it passes from a clean checkout.
- [ ] Configure an initial Windows packaged build and packaged-app smoke test. Verify the packaged app opens without a developer server.
- [ ] Add `README.md` with exact install, start, build, test, and verify commands. Verify a fresh clone can follow the instructions.
- [ ] Add `AGENTS.md` with repo commands, architecture boundaries, and the rule that each checklist item should be one commit. Verify commands match `package.json`.
- [ ] Add `ARCHITECTURE.md` describing main, preload, renderer, and local-data ownership. Verify it does not commit to unvalidated persistence details.

## 2. Runnable UI Shell

- [ ] Create the application frame with placeholder Library, Agenda, Annotated, Bookmarks, and Settings destinations. Verify each destination is reachable.
- [ ] Add design-language CSS tokens for typography, spacing, surfaces, text, borders, accent, focus, and status colors. Verify a visual smoke test in light mode.
- [ ] Add dark-theme token values driven by the system color scheme. Verify a visual smoke test in dark mode.
- [ ] Build a responsive navigation rail that collapses below the documented breakpoints. Verify at wide, medium, and narrow window widths.
- [ ] Add accessible names, tooltips, focus states, and minimum target sizing to shell controls. Verify keyboard navigation and an accessibility smoke test.
- [ ] Add a renderer error boundary and a main-process startup error log. Verify a forced renderer error shows a recoverable error surface.

## 3. Early PDF and Ink Vertical Slice

- [ ] Add deterministic one-page, multi-page, text-heavy, and image-heavy PDF test fixtures. Verify the fixtures open in an external PDF reader.
- [ ] Add a native Open PDF command through a narrow preload API. Verify canceling and selecting a local PDF both behave correctly.
- [ ] Render one selected PDF page with PDF.js. Verify text-heavy and image-heavy sample PDFs render visibly.
- [ ] Render all PDF pages as a continuous vertical roll. Verify scrolling through a multi-page PDF remains responsive.
- [ ] Add per-page transparent SVG overlays sized to the PDF.js viewport. Verify overlays remain aligned after window resize.
- [ ] Add normalized page-coordinate helpers with unit tests. Verify points round-trip between screen and page coordinates.
- [ ] Centralize pointer-tool resolution for mouse, pen tip, hardware eraser contact, and touch. Verify unit tests cover `button === 5` and `buttons & 32`.
- [ ] Add pointer diagnostics showing raw pointer fields, resolved tool, active tool, and rendered tool. Verify diagnostics update during mouse, pen, eraser, and touch input.
- [ ] Implement black freehand strokes for pen and explicit mouse drawing mode. Verify strokes stay on the page where they began.
- [ ] Apply pen pressure to rendered stroke width. Verify pressure changes are visible on the target pen device.
- [ ] Route one-finger touch to document scrolling without drawing. Verify touch scroll and pen drawing separately on the target device.
- [ ] Implement whole-stroke erasing with the hardware eraser and toolbar eraser. Verify both paths erase the intended stroke.
- [ ] Add session-only undo and redo for stroke add and erase actions. Verify reducer unit tests and toolbar behavior.
- [ ] Add a compact Pen, Eraser, Undo, and Redo toolbar. Verify controls work with mouse, keyboard, and pen.
- [ ] Keep PDF page status visible while loading, rendering, or failing. Verify a forced PDF error produces a readable page-level state.
- [ ] Run `npm run verify` and manually test a multi-page PDF on the target Windows pen device. Verify Electron, PDF.js, pen, eraser, touch-scroll, and cursor findings are recorded in `ARCHITECTURE.md`.
- [ ] Gate: accept the target-device PDF and ink vertical slice before beginning durable annotation plumbing. Verify the acceptance result is recorded in `ARCHITECTURE.md`.

## 4. Persistence Capability Spike

- [ ] Define minimal TypeScript entities for conference, talk, deck, slide, pen stroke, text note, and view state. Verify model unit tests cover stable IDs and annotation counts.
- [ ] Spike Electron `42.3.2` access to SQLite from the main process without exposing database access to the renderer. Verify create, write, read, transaction, and restart behavior in a packaged build.
- [ ] Document the accepted local-data approach and fallback in `ARCHITECTURE.md`. Verify the decision cites the packaged-build spike result.
- [ ] Create a versioned database initialization and migration runner. Verify a fresh database and one fixture upgrade both succeed.
- [ ] Add repository methods for conferences and talks. Verify CRUD and transaction tests.
- [ ] Add repository methods for decks, slides, annotations, and view state. Verify CRUD and cascade-behavior tests.
- [ ] Add debounced automatic annotation saves through a narrow preload API. Verify rapid stroke updates persist without one IPC call per pointer event.
- [ ] Restore strokes and slide view state after app restart. Verify an Electron integration test closes and reopens the app.
- [ ] Add a visible Saving/Saved/Error status driven by save completion. Verify a forced write failure does not discard in-memory work.

## 5. Conference Library with Fixtures

- [ ] Add deterministic fixture data for a small event and a large nested event. Verify fixture validation tests pass.
- [ ] Build the empty Library view with event URL input and Open event action. Verify invalid URLs remain visible with an inline error.
- [ ] Add an import-fixture developer command that stores a conference locally. Verify the imported event appears after restart.
- [ ] Build the populated Library list ordered by most recently opened. Verify title, dates, host, last-opened time, annotation count, and cache status render.
- [ ] Open a stored fixture event from the Library into an Agenda placeholder. Verify returning to Library preserves the event.
- [ ] Add conference deletion confirmation and local-data cascade deletion. Verify cancel preserves data and confirm removes all conference-owned data.

## 6. Indico Event Integration

- [ ] Add an Indico event URL parser and canonical event identity helper. Verify unit tests cover accepted, rejected, and trailing-path URLs.
- [ ] Add a main-process HTTPS client limited to validated Indico origins and expected response sizes. Verify mocked success, timeout, HTTP error, and invalid JSON cases.
- [ ] Fetch one public Indico event into a raw-response fixture without storing credentials. Verify the fixture can be replayed offline.
- [ ] Map the raw Indico response into conference, hierarchy, talk, speaker, time, contribution URL, and material entities. Verify mapper tests cover missing and inconsistent fields.
- [ ] Connect Open event to fetch, map, store, and open a public event. Verify a real public event reopens from the local cache after restart.
- [ ] Show fetch progress and actionable inline failures while existing conferences remain usable. Verify an offline fetch failure preserves stored data.
- [ ] Add API-key entry and secure local credential storage with Electron `safeStorage` for private events. Verify logs and stored conference records never contain the plaintext key.

## 7. Agenda Fundamentals

- [ ] Build a simple single-column agenda list from stored data before the two-dimensional canvas. Verify a large fixture remains usable.
- [ ] Add day selection and preserve the approximate time position per day. Verify switching away and back restores position.
- [ ] Add talk rows with time, title, speaker, bookmark, PDF state, and annotated-slide count. Verify states use text plus icons.
- [ ] Add talk details with metadata, materials, and Open slides action. Verify wide and narrow presentations are usable.
- [ ] Add bookmark persistence and Bookmarks destination. Verify bookmark state survives restart.
- [ ] Add Annotated destination from derived annotation counts. Verify it updates after adding and removing annotations.
- [ ] Add filters for All, Bookmarked, Annotated, and Slides available. Verify filters preserve the selected day.
- [ ] Add event-wide search over title, speaker, session, contribution ID, and keywords. Verify selecting a result opens the correct talk and day.
- [ ] Replace the single-column agenda with the continuous two-dimensional day canvas. Verify time is vertical and simultaneous sessions are horizontal.
- [ ] Pin the time gutter and add shared horizontal and vertical scrolling. Verify wheel, Shift+wheel, touch, trackpad, scrollbars, and arrow keys.
- [ ] Add responsive session-column widths and full-width shared agenda items. Verify wide, medium, portrait, and ten-session fixture layouts.
- [ ] Run a keyboard and screen-reader agenda audit. Verify logical focus order, explicit status names, and no color-only information.

## 8. Slide Deck Download and Viewer

- [ ] Identify annotatable PDF materials and keep non-PDF materials visible only in talk details. Verify mixed-material fixture behavior.
- [ ] Add a multiple-PDF chooser and remember the selected default deck per talk. Verify the choice survives restart.
- [ ] Download a selected PDF only when Open slides is used. Verify the agenda fetch does not pre-download PDFs.
- [ ] Store cached PDFs under the conference-owned app-data directory using stable deck IDs. Verify cached files reopen offline.
- [ ] Add download progress, cancellation, retry, and clear failure states. Verify a simulated interrupted download leaves no valid-looking partial cache.
- [ ] Connect cached deck bytes to the validated continuous PDF and ink viewer. Verify opening a talk reaches an annotatable deck.
- [ ] Add current slide, total slides, jump-to-slide, zoom, and pan. Verify view state survives leaving and reopening the talk.
- [ ] Add annotated-slide markers in the slide navigator. Verify markers update after add, erase, undo, and redo.

## 9. Durable Pen and Text Annotation

- [ ] Persist normalized pen strokes with stable event, talk, deck, and page identity. Verify strokes survive restart and resize.
- [ ] Persist undo and redo actions without corrupting the current annotation state. Verify restart after undo and after redo.
- [ ] Add Text mode that places a black editable note from mouse click or pen tap. Verify finger touch continues to navigate.
- [ ] Add text-note edit and delete behavior. Verify changes persist after restart.
- [ ] Add text-note drag positioning in normalized page coordinates. Verify position survives resize, zoom, and restart.
- [ ] Include text-note changes in undo and redo. Verify mixed pen/text history tests.
- [ ] Update talk and slide annotation counts from active annotations only. Verify deleting the last annotation clears all related indicators.
- [ ] Add an in-page pen/eraser marker if target-device cursor testing shows the native stylus cursor is insufficient. Verify mouse cursor behavior remains unchanged.

## 10. Export Vertical Slice

- [ ] Render one annotated slide to a PNG containing the PDF page, pen strokes, and text notes. Verify output dimensions and a visual fixture.
- [ ] Build deterministic Markdown for one annotated talk and embedded PNG data URL. Verify a snapshot test.
- [ ] Export only annotated talks and annotated slides for the current event. Verify unannotated content is absent.
- [ ] Add conference and talk metadata plus original Indico links to export. Verify missing optional metadata does not break output.
- [ ] Add the Windows save dialog and write one self-contained Markdown file. Verify the exported file opens after restart and offline.
- [ ] Warn and produce no file when the event has no annotated slides. Verify no empty file is created.
- [ ] Add export progress and cancellation. Verify cancellation leaves no partial final file.
- [ ] Add Open file location after successful export. Verify the action opens the containing folder.
- [ ] Expose Export notes from both Agenda and Slide Notes views. Verify both actions export the entire current event.

## 11. Refresh, Offline, and Reliability

- [ ] Add manual agenda refresh that keeps old data usable until new data is committed. Verify a failed refresh leaves the old agenda intact.
- [ ] Reconcile refreshed talks by stable Indico identity while preserving annotations for changed, moved, renamed, or removed talks. Verify reconciliation fixture tests.
- [ ] Surface removed or changed upstream items without hiding annotated work. Verify affected talks remain exportable.
- [ ] Detect newly available PDF materials during refresh. Verify they appear without blocking or notifying the user.
- [ ] Detect changed PDF material when annotations exist and require an explicit replacement decision. Verify decline preserves the cached deck and annotations.
- [ ] Add app launch with an externally supplied Indico URL. Verify valid, invalid, and already-open event URLs.
- [ ] Add crash-safe writes using transactions and atomic file replacement where applicable. Verify forced interruption tests do not corrupt stored work.
- [ ] Add a local-data location display and Open data folder action in Settings. Verify it opens the correct user-specific directory.
- [ ] Add structured logs that exclude credentials and annotation content. Verify failure logs are useful and privacy-safe.

## 12. V1 Packaging and Acceptance

- [ ] Configure a Windows installer that requires no administrator privileges for normal use. Verify install, launch, upgrade, and uninstall on a clean Windows account.
- [ ] Ensure packaged builds include PDF.js assets, database support, migrations, and renderer assets. Verify the packaged app passes the smoke suite.
- [ ] Add a packaged-app end-to-end test for Library, cached event, deck open, annotation restore, and export. Verify it passes on Windows.
- [ ] Measure a large real-event agenda and a long PDF deck. Verify navigation remains responsive enough for live-talk use.
- [ ] Audit all primary workflows at wide landscape, narrow landscape, and portrait sizes. Verify controls remain reachable and slides remain the focus.
- [ ] Audit keyboard, touch, pen, mouse, screen reader, contrast, text scaling, and reduced motion. Verify documented accessibility requirements.
- [ ] Run the complete V1 acceptance flow: open an Indico event, find a talk, open slides, draw, add text, restart, and export one self-contained Markdown file. Verify every step succeeds in sequence.
- [ ] Run `npm run verify` and the target-device manual checklist against the final pinned Electron version. Verify results and known limitations are recorded.
